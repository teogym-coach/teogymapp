# 회원앱 로그인 흐름

## 1. 회원 문서 준비
관리자 앱에서 회원을 등록할 때 `members/{memberId}` 문서에 회원 이메일을 저장합니다. 회원앱은 로그인한 Firebase Authentication 사용자의 UID(`memberUid`)를 먼저 조회하고, 없으면 이메일로 `members` 문서를 한 번 더 찾습니다.

## 2. 회원앱 계정 생성
관리자 회원 상세 화면의 **[회원앱 계정 생성]** 버튼을 사용합니다.

1. 저장된 회원 이메일을 기준으로 Firebase Authentication 계정을 생성합니다.
2. 임시 비밀번호를 생성해 Auth 계정 생성에만 사용합니다.
3. 즉시 비밀번호 재설정 메일을 발송합니다.
4. 회원은 메일 링크에서 직접 비밀번호를 설정합니다.
5. `members` 문서에 생성 여부(`memberUid`, `memberAppAccountCreatedAt`)를 저장해 중복 생성을 막습니다.

## 3. 회원 로그인
회원이 `/member` 또는 `?app=member` 화면에서 이메일과 비밀번호로 로그인합니다.

- Firebase Authentication 계정이 없으면 로그인 화면에 “회원 이메일은 저장되어 있지만 Firebase Authentication 계정이 없습니다. 대표에게 회원앱 계정 생성을 요청해주세요.” 안내가 표시됩니다.
- 비밀번호가 틀렸거나 다른 인증 오류가 있으면 일반 로그인 실패 안내가 표시됩니다.

## 4. 프로필 로딩
로그인 후 `getMemberAppProfile()`을 호출합니다.

1. 현재 Auth UID와 `members.memberUid`가 일치하는 문서를 찾습니다.
2. UID 매칭 문서가 없으면 로그인 이메일과 `members.email`이 일치하는 문서를 찾습니다.
3. 둘 다 없으면 “회원 정보를 찾을 수 없습니다. 대표에게 문의해주세요.” 화면을 표시합니다.
4. 호출 실패 또는 후속 데이터 로딩 실패 시 로딩을 종료하고 에러 화면을 표시합니다.
5. 로그인 후 5초 이상 응답이 없으면 타임아웃 에러 화면을 표시합니다.

## 5. 회원앱 데이터 로딩
프로필 문서를 찾으면 공개 수업일지, 바디체크, 영양, 체크인, 메시지, 온보딩 정보를 병렬로 불러옵니다. 이 단계도 5초 타임아웃을 적용해 무한 로딩을 방지합니다.
